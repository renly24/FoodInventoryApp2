# 食品管理アプリ 機能分析レポート

## 1. アプリケーション概要

このアプリケーションは、家庭の食品在庫、買い物、レシピ、食事記録を統合的に管理するためのモバイルアプリケーションです。ユーザーは日々の食品管理を効率化し、無駄を減らすことができます。

## 2. 主要機能

このアプリは、以下の主要なタブ（機能）で構成されています。

### 2.1. ホーム画面 (ダッシュボード)

アプリ起動時のメイン画面であり、各機能の概要を一覧できるダッシュボードとして機能します。

- **サマリー表示:**
  - **買い物リスト:** 現在のアイテム数と概算の合計金額を表示します。
  - **在庫状況:** 在庫アイテムの総数、評価額、および在庫が少なくなっているアイテムを警告します。
  - **レシピ:** 登録されているレシピの総数を表示します。
  - **食事記録:** 最近作られた食事の履歴を表示します。
- **ナビゲーション:** 各機能（在庫、買い物、レシピ、食事）の詳細画面へ素早く移動するためのカード型リンクを提供します。

### 2.2. 在庫管理

家にある食品や日用品の在庫を管理する機能です。

- **在庫リスト:** 在庫アイテムを一覧で表示します。
- **アイテム登録・編集:**
  - アイテム名、数量、単位、価格、カテゴリ（食品、日用品など）、メモ、優先度を登録・編集できます。
  - バリデーション機能があり、必須項目（名前、数量）がチェックされます。
- **数量の自動更新:** 同じアイテムが追加された場合、既存のアイテムの数量が自動的に加算され、重複登録を防ぎます。

### 2.3. 買い物リスト

購入予定のアイテムを管理する機能です。

- **リスト作成:** 買い物に行く前に必要なアイテムをリストアップします。
- **アイテム登録・編集:**
  - アイテム名、数量、単位、予定単価、カテゴリ、メモ、優先度を登録できます。
  - 購入済みかどうかを切り替えるステータス管理機能があります。
- **重複チェック:** リストに同じアイテムを追加しようとすると、アラートで通知し、数量を更新するかどうかを選択できます。

### 2.4. レシピ管理

作りたい料理のレシピを保存・管理する機能です。

- **レシピ登録:** 料理名と、それに必要な材料（名前、量、単位）を登録します。
- **在庫連携:** レシピの材料は、在庫管理機能に登録されているアイテムと連携させることができます。これにより、レシピを作るために必要な材料が在庫にどれだけあるかを確認できます（`RecipeTab` コンポーネントで実装）。

### 2.5. 食事記録

日々の食事を記録する機能です。

- **食事の記録:** 作成した食事を日付と共に記録します。
- **在庫連携による自動更新:**
  - レシピや在庫アイテムから食事を記録すると、使用した材料の分だけ在庫リストの数量が自動的に減少します。
  - 在庫が不足している場合は、買い物リストにアイテムを自動で追加する機能も含まれている可能性があります（`MealTab` コンポーネントで実装）。

### 2.6. プロフィール・設定

ユーザーアカウント情報やアプリの設定を管理する画面です。

- **ユーザー情報:** 現在ログインしているユーザーのメールアドレスなどを表示します。
- **ログアウト:** アプリからログアウトする機能を提供します。
- **各種設定（プレースホルダー）:** アカウント情報、通知設定、プライバシー設定などの項目が用意されていますが、現時点ではダミーとなっています。

## 3. データ管理

- **ローカルストレージ:** `AsyncStorage` を利用して、在庫、買い物リスト、レシピ、食事記録などのデータをデバイス上に永続的に保存します。
- **Firebase連携:** ユーザー認証機能にFirebase Authenticationが使用されています。将来的にはFirestoreなどを利用したデータ同期機能の実装も考えられます。

## 4. 結論

このアプリは、単なるリスト管理ツールに留まらず、在庫、買い物、レシピ、食事記録の各機能を連携させることで、包括的な食品管理ソリューションを提供します。特に、食事記録と連動した在庫の自動更新機能は、ユーザーの手間を大幅に削減し、食品ロスを防ぐのに役立ちます。

## 5. 技術スタック

プロジェクト全体および各コンポーネントで使用されている主要な技術スタックは以下の通りです。

### 5.1. 全体

- **Core Framework:** React Native (v0.79.5) with Expo (SDK 53)
- **Language:** TypeScript
- **Routing:** Expo Router (v5) - ファイルベースのルーティングを実現
- **UI & Styling:**
  - React Native-reanimated - 高度なアニメーション
  - React Native-gesture-handler - ジェスチャー操作
  - Expo-blur, Expo-linear-gradient - UIの視覚効果
  - スタイルはStyleSheetで定義され、`styles/theme.ts` で共通のテーマ（色、スペーシング等）を管理

### 5.2. 主要コンポーネント・機能別

- **認証 (Login/Profile):**
  - `firebase/auth`: Firebase Authenticationによるメール/パスワード認証
  - `contexts/AuthContext.tsx`: React Context APIを利用して、アプリ全体で認証状態（ユーザー情報）を管理

- **データ永続化 (Storage):**
  - `@react-native-async-storage/async-storage`: 在庫、買い物リスト、レシピ、食事記録などのデータをデバイスのローカルストレージにJSON形式で保存

- **UIコンポーネント (`components/`):**
  - **カスタムUI (`components/ui/`):**
    - `Card`, `Chip`, `CustomModal`, `FAB` など、アプリ全体で再利用される基本的なUI部品
    - `IconSymbol`: `expo-symbols` を利用し、AppleのSF Symbolsをアプリ内で使用
  - **機能別コンポーネント:**
    - `InventoryTab`, `ShoppingTab`, `RecipeTab`, `MealTab`: 各タブ画面のメインロジックとUIをカプセル化
    - `CustomPicker`: `@react-native-picker/picker` をラップしたカスタムの選択コンポーネント

- **ユーティリティ:**
  - `uuid`: 在庫アイテムなどに一意のIDを付与するために使用
  - `expo-haptics`: ボタンタップ時などに触覚フィードバックを提供

### 5.3. 開発・ビルド

- **Linter:** ESLint with `eslint-config-expo`
- **Build & Deployment:** EAS CLI (`eas-cli`)
